# Recherche documentaire — Guide de référence

> **Quand charger ce fichier :** lorsqu'un étudiant demande de l'aide pour trouver des sources académiques, évaluer la fiabilité d'un document, naviguer dans les bases de données universitaires, ou construire une stratégie de recherche bibliographique. Applicable du lycée au master.

---

## 1. Opérateurs de recherche avancée

### Google Scholar

Google Scholar est souvent le point de départ. Voici les opérateurs essentiels :

| Opérateur | Fonction | Exemple |
|-----------|----------|---------|
| `"..."` (guillemets) | Recherche exacte d'une expression | `"gouvernance d'entreprise"` |
| `author:` | Filtrer par auteur | `author:Piketty inégalités` |
| `intitle:` | Mot présent dans le titre | `intitle:"intelligence artificielle" éthique` |
| `site:` | Limiter à un domaine | `site:cairn.info comptabilité IFRS` |
| `OR` | Chercher l'un ou l'autre terme | `"RSE" OR "responsabilité sociétale"` |
| `-` | Exclure un terme | `blockchain -bitcoin` |
| Plage de dates | Via le menu latéral gauche | Sélectionner « Depuis 2020 » ou personnaliser |

**Astuce :** dans les paramètres de Google Scholar, cocher « Afficher les liens pour importer dans BibTeX » — cela fait gagner un temps considérable pour la bibliographie.

### Cairn.info

Cairn est la principale plateforme francophone pour les sciences humaines et sociales, la gestion et le droit.

- **Recherche par revue** : aller dans « Revues » → choisir la discipline (ex. : *Comptabilité Contrôle Audit*, *Revue française de gestion*, *Droit social*)
- **Filtres disponibles** : discipline, année, type de publication, accès libre uniquement
- **Recherche par discipline** : utile pour explorer un champ (ex. : Gestion → sous-catégorie « Comptabilité »)
- **Bouton « Citer »** : génère directement la référence en APA, ce qui évite les erreurs

### HAL (archives ouvertes françaises)

HAL (`hal.science`) est le dépôt national d'archives ouvertes. Tout y est en accès libre.

- **Filtres** : par discipline, type de document (article, thèse, communication, HDR), année, laboratoire
- **Recherche avancée** : combiner des champs (titre, auteur, résumé, mots-clés)
- **Intérêt principal** : accéder gratuitement à des articles qui seraient payants ailleurs (les auteurs y déposent souvent la version « postprint »)
- **TEL** (`theses.hal.science`) : spécifiquement pour les thèses de doctorat françaises

### Persée

Persée (`persee.fr`) donne accès aux numéros anciens de revues françaises numérisées.

- Idéal pour les recherches historiques ou l'évolution d'un concept
- Contenus en accès libre intégral
- Exemple : retrouver un article fondateur des *Annales* ou de la *Revue économique* des années 1960-1990

### DALLOZ / Légifrance (droit et comptabilité)

| Plateforme | Usage | Accès |
|------------|-------|-------|
| **Légifrance** (`legifrance.gouv.fr`) | Textes de loi, codes, jurisprudence, JORF | Gratuit |
| **Dalloz.fr** | Encyclopédies juridiques, revues, commentaires d'arrêts | Via bibliothèque universitaire |
| **Doctrine.fr** | Jurisprudence avec outils d'analyse | Accès limité gratuit + premium |

**Pour la comptabilité** : chercher les textes du PCG sur Légifrance, les normes IFRS via le site de l'ANC (`anc.gouv.fr`), et les analyses doctrinales sur Dalloz ou Cairn.

### PubMed (santé, biologie)

- Accès : `pubmed.ncbi.nlm.nih.gov` (gratuit)
- Utiliser les **MeSH terms** (vocabulaire contrôlé) pour des recherches précises
- Filtre « Free full text » pour accéder directement aux articles en accès libre
- Opérateurs : `AND`, `OR`, `NOT`, champs `[ti]` (titre), `[au]` (auteur), `[mh]` (MeSH)

### IEEE Xplore / ACM Digital Library (informatique)

| Plateforme | Spécialité | Accès |
|------------|-----------|-------|
| **IEEE Xplore** (`ieeexplore.ieee.org`) | Électronique, réseaux, IA, systèmes embarqués | Via BU |
| **ACM DL** (`dl.acm.org`) | Informatique théorique et appliquée, IHM, algorithmes | Via BU |

- Les deux permettent la recherche par conférence (important en info, où les conférences comptent autant que les revues)
- **DBLP** (`dblp.org`) : moteur de recherche bibliographique spécialisé en informatique, utile pour retrouver toutes les publications d'un auteur

## 2. Comment lire un article académique

### Ordre de lecture efficace

Ne jamais lire un article de la première à la dernière page. Procéder ainsi :

```
1. Abstract (résumé)        → De quoi parle l'article ? Ça m'intéresse ?
2. Conclusion               → Quels résultats ? Quelle contribution ?
3. Introduction              → Quel est le contexte et la question de recherche ?
4. Méthode                   → Comment ont-ils fait ? (échantillon, terrain, outils)
5. Résultats                 → Que montrent les données concrètement ?
6. Discussion                → Comment les auteurs interprètent-ils leurs résultats ?
```

Cette méthode permet de décider en **5 minutes** si un article mérite une lecture approfondie, au lieu d'y passer une heure pour réaliser qu'il n'est pas pertinent.

### Ce qu'il faut extraire

Pour chaque article retenu, identifier :

- **La question de recherche** : que cherchent-ils à démontrer ou comprendre ?
- **La méthodologie** : qualitative (entretiens, étude de cas) ou quantitative (régression, enquête) ? Quel échantillon ?
- **Les résultats clés** : 2-3 conclusions principales, en une phrase chacune
- **Les limites** : ce que les auteurs reconnaissent eux-mêmes comme faiblesses (souvent en fin de discussion)
- **Les pistes de recherche** : utiles si on travaille sur un sujet proche

### Prise de notes pendant la lecture

Utiliser un code couleur (sur PDF ou papier) :

| Couleur | Signification |
|---------|---------------|
| Jaune | Définitions et concepts clés |
| Vert | Résultats et conclusions importantes |
| Bleu | Éléments méthodologiques |
| Rouge | Citations à reprendre dans mon travail |
| Orange | Points de désaccord ou limites |

## 3. Types de sources

### Source primaire

C'est le matériau de première main, la source originale.

| Discipline | Exemples de sources primaires |
|------------|-------------------------------|
| Droit | Texte de loi, arrêt de la Cour de cassation, directive européenne |
| Gestion/Compta | Rapport annuel d'entreprise, données financières brutes, norme IFRS |
| Informatique | Code source, documentation technique officielle, RFC |
| SHS | Entretien retranscrit, archives, données d'enquête (INSEE) |

### Source secondaire

Elle analyse, commente ou synthétise des sources primaires.

- **Manuels** : bon point de départ, mais insuffisant comme seule source en master
- **Méta-analyses** : synthèse statistique de plusieurs études (très valorisée)
- **Revues de littérature** : synthèse narrative ou systématique d'un champ de recherche
- **Commentaires d'arrêts** (droit) : analyse doctrinale d'une décision de justice

### Littérature grise

Documents produits en dehors des circuits d'édition classiques :

- Rapports d'institutions (Cour des comptes, INSEE, OCDE, Banque de France)
- Mémoires de master et thèses (sur TEL, DUMAS)
- Working papers / documents de travail (sur SSRN, NBER, HAL)
- Rapports d'audit, documents internes d'entreprise

**Attention :** la littérature grise n'est pas systématiquement relue par des pairs, mais elle peut être très riche en données. Toujours vérifier l'institution émettrice.

### Sources à éviter en citation directe

| Source | Pourquoi l'éviter | Alternative |
|--------|-------------------|-------------|
| Wikipédia | Non relue par des pairs, contenu modifiable | Utiliser comme point de départ, puis remonter aux sources citées en bas de page |
| Blogs non-experts | Pas de garantie de rigueur | Chercher l'article académique dont le blog s'inspire |
| Sites de cours (Cours-de-droit.net, etc.) | Simplifications, erreurs possibles, pas d'auteur identifié | Consulter le manuel de référence du domaine |
| Articles de presse généraliste | Pas de méthodologie, risque de biais | Acceptable pour illustrer un contexte, pas comme preuve scientifique |

## 4. Évaluation d'une source — méthode CRAAP adaptée au contexte français

La méthode CRAAP (Currency, Relevance, Authority, Accuracy, Purpose) est un cadre anglo-saxon qu'on adapte ici aux réalités académiques françaises.

### Currency (Actualité)

- **Vérifier la date de publication.** Un article de 2005 sur le cloud computing est obsolète ; un article de 2005 sur la théorie de l'agence reste pertinent.
- **Règle générale :** pour les domaines à évolution rapide (informatique, fiscalité, droit positif), privilégier les sources de moins de 5 ans. Pour les fondements théoriques, les classiques restent valables.
- **Attention aux éditions :** un manuel peut dater de 2023 mais reprendre un contenu inchangé depuis 2010. Vérifier la date de dernière mise à jour réelle.

### Relevance (Pertinence)

- La source répond-elle directement à ma problématique ou à une de mes sous-questions ?
- Est-ce que je l'utilise pour un concept, une donnée, une méthode, ou juste pour « remplir » la bibliographie ?
- **Test pratique :** si on ne peut pas expliquer en une phrase pourquoi cette source est dans la bibliographie, elle n'y a probablement pas sa place.

### Authority (Autorité)

Qui est l'auteur et où publie-t-il ?

| Critère | Ce qu'il faut vérifier |
|---------|----------------------|
| Auteur | Est-il chercheur ? Rattaché à quelle université ou labo ? A-t-il d'autres publications sur le sujet ? |
| Revue | Est-elle classée par le **HCERES** (toutes disciplines) ? Par la **FNEGE** (gestion) ? Par le **CNU** (droit) ? |
| Éditeur | Éditeur universitaire reconnu (PUF, Dalloz, LGDJ, De Boeck, Dunod) vs. autoédition |
| Institution | Rapport de la Cour des comptes ou de l'INSEE = forte autorité. Rapport d'un think tank = vérifier l'orientation |

**Classement HCERES des revues :** le Haut Conseil de l'évaluation de la recherche et de l'enseignement supérieur classe les revues scientifiques françaises et internationales. Une revue classée HCERES a un processus de relecture par les pairs. Le classement FNEGE (rang 1 à 4) est spécifique à la gestion.

### Accuracy (Exactitude)

- L'article décrit-il clairement sa **méthodologie** ? (échantillon, protocole, outils statistiques)
- Les données sont-elles **vérifiables** ? (sources citées, bases de données identifiées)
- L'article est-il **peer-reviewed** (relu par des pairs) ? C'est la garantie minimale de rigueur.
- Les résultats sont-ils **cohérents** avec la littérature existante ? S'ils divergent, l'auteur explique-t-il pourquoi ?

### Purpose (Objectif)

Quel est le but du texte ?

| Objectif | Indice | Fiabilité |
|----------|--------|-----------|
| Informatif / scientifique | Méthodologie rigoureuse, ton neutre, limites reconnues | Élevée |
| Pédagogique | Manuel, cours en ligne | Moyenne (simplifications possibles) |
| Commercial | Livre blanc d'un cabinet de conseil, étude d'un éditeur de logiciel | Faible (biais d'intérêt) |
| Militant / idéologique | Think tank orienté, association de promotion | Faible (sélection des données) |

**Règle d'or :** une source peut être biaisée et quand même utile — à condition de le signaler explicitement dans le travail. Écrire « Selon le rapport du Medef (2023), qui représente le point de vue patronal, ... » est honnête intellectuellement.

## 5. Accès aux ressources via bibliothèque universitaire

### Accès distant via ENT

La plupart des universités françaises donnent accès aux bases de données payantes depuis chez soi :

1. Se connecter à l'**ENT** (Espace Numérique de Travail) de l'université
2. Aller dans la rubrique **Bibliothèque** ou **Documentation**
3. Accéder au portail documentaire (souvent propulsé par un outil comme **Primo**, **Summon** ou **EDS**)
4. Les bases de données reconnaissent automatiquement l'appartenance à l'université

### Proxy / VPN universitaire

Si l'accès via l'ENT ne fonctionne pas :

- Configurer le **proxy** de la BU dans le navigateur (instructions généralement sur le site de la BU)
- Certaines universités proposent un **VPN** (ex. : `vpn.univ-paris1.fr`) qui simule une connexion depuis le campus
- L'extension **EZproxy Redirect** peut automatiser la redirection

### Prêt inter-bibliothèques (PEB)

Quand un ouvrage ou un article n'est pas disponible dans sa BU :

- Demander un **PEB** via le site de la BU (souvent gratuit pour les articles, quelques euros pour les ouvrages)
- Délai : généralement 1 à 2 semaines
- **SUDOC** (`sudoc.abes.fr`) : catalogue collectif des BU françaises — permet de localiser un ouvrage dans n'importe quelle BU de France

### Bases de données accessibles via la BU

| Base | Contenu | Discipline |
|------|---------|-----------|
| **ScienceDirect** (Elsevier) | Articles de revues, chapitres d'ouvrages | Toutes disciplines |
| **SpringerLink** | Articles, ouvrages, séries | Sciences, informatique, gestion |
| **Wiley Online Library** | Articles de revues | Toutes disciplines |
| **JSTOR** | Archives de revues (souvent anciens numéros) | SHS principalement |
| **Cairn.info** | Revues et ouvrages francophones | SHS, gestion, droit |
| **Dalloz** | Encyclopédies, revues juridiques | Droit |
| **EBSCOhost** | Multibases (Business Source, etc.) | Gestion, SHS |
| **Statista** | Données statistiques et études de marché | Gestion, marketing |

### Accès libre (Open Access)

Pas besoin d'être rattaché à une université pour y accéder :

| Ressource | Contenu |
|-----------|---------|
| **HAL** (`hal.science`) | Archives ouvertes françaises, toutes disciplines |
| **arXiv** (`arxiv.org`) | Prépublications en maths, physique, informatique, économie |
| **DOAJ** (`doaj.org`) | Répertoire de revues en accès libre (vérifier la qualité) |
| **Unpaywall** | Extension navigateur qui détecte automatiquement les versions libres d'un article |
| **Sci-Hub** | ⚠️ Existe mais illégal en France — ne pas recommander |
| **DUMAS** (`dumas.ccsd.cnrs.fr`) | Mémoires de master en accès libre |
| **OpenEdition** (`openedition.org`) | Livres et revues en SHS, en accès libre ou freemium |

## 6. Mots-clés multilingues — stratégie de recherche

### Chercher en anglais ET en français

La majorité de la littérature scientifique mondiale est en anglais. Se limiter au français, c'est passer à côté de 80 % des sources.

**Méthode :** pour chaque concept clé, préparer une version française et une version anglaise.

| Français | Anglais |
|----------|---------|
| Gouvernance d'entreprise | Corporate governance |
| Responsabilité sociétale des entreprises | Corporate social responsibility (CSR) |
| Contrôle de gestion | Management control / Management accounting |
| Audit légal | Statutory audit |
| Droit des sociétés | Company law / Corporate law |
| Apprentissage automatique | Machine learning |
| Traitement automatique du langage | Natural language processing (NLP) |
| Données personnelles | Personal data / Privacy |
| Contrat de travail | Employment contract |
| Normes comptables internationales | International Financial Reporting Standards (IFRS) |
| Chaîne de blocs | Blockchain |
| Inégalités scolaires | Educational inequalities |

### Utiliser les thesaurus disciplinaires

Un thesaurus est un vocabulaire normalisé propre à une discipline. Il permet de trouver le « bon mot » utilisé dans les bases de données.

- **RAMEAU** : thesaurus de la BnF, utilisé dans les catalogues de BU françaises
- **MeSH** : pour les sciences de la santé (PubMed)
- **JEL codes** : classification en économie (ex. : M41 = Accounting)
- **ACM Computing Classification System** : pour l'informatique

### Technique de la « boule de neige »

C'est la méthode la plus efficace pour construire une bibliographie solide :

```
1. Trouver UN bon article récent (< 3 ans) sur son sujet
2. Regarder sa BIBLIOGRAPHIE → identifier les auteurs et articles clés qu'il cite
3. Lire ces articles clés
4. Pour chaque article clé, vérifier sur Google Scholar « Cité par »
   → trouver des articles PLUS RÉCENTS qui le citent
5. Répéter jusqu'à atteindre une saturation (on retombe toujours sur les mêmes noms)
```

Cette technique combine la recherche **en amont** (remonter les références) et **en aval** (trouver qui cite un article donné). Elle permet de cartographier un champ de recherche en quelques heures.

## 7. Fiche de lecture type

Utiliser ce modèle pour chaque source retenue dans la bibliographie. Remplir au fur et à mesure de la lecture.

```
╔══════════════════════════════════════════════════════════════╗
║                      FICHE DE LECTURE                       ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║  Référence complète (APA 7) : ...                           ║
║                                                              ║
║  Type de source : article / ouvrage / rapport / thèse /     ║
║                   chapitre / communication / autre           ║
║                                                              ║
║  Revue / Éditeur : ...                                      ║
║  Classement (si applicable) : HCERES / FNEGE rang ... /     ║
║                                non classée                   ║
║                                                              ║
║  ─────────────────────────────────────────────────────────── ║
║                                                              ║
║  Question de recherche : ...                                 ║
║                                                              ║
║  Cadre théorique : ...                                       ║
║                                                              ║
║  Méthodologie :                                              ║
║    - Type : qualitative / quantitative / mixte               ║
║    - Échantillon / terrain : ...                             ║
║    - Outils : ...                                            ║
║                                                              ║
║  Résultats clés :                                            ║
║    1. ...                                                    ║
║    2. ...                                                    ║
║    3. ...                                                    ║
║                                                              ║
║  Citations utiles (avec pages) :                             ║
║    - « ... » (p. ...)                                        ║
║    - « ... » (p. ...)                                        ║
║                                                              ║
║  Limites identifiées par l'auteur : ...                      ║
║                                                              ║
║  ─────────────────────────────────────────────────────────── ║
║                                                              ║
║  Lien avec ma problématique : ...                            ║
║                                                              ║
║  Évaluation CRAAP :                                          ║
║    Actualité : /5   Pertinence : /5   Autorité : /5         ║
║    Exactitude : /5   Objectif : /5                           ║
║                                                              ║
║  À relier avec (autres fiches) : ...                         ║
║                                                              ║
╚══════════════════════════════════════════════════════════════╝
```

### Version simplifiée (pour un premier tri rapide)

```
FICHE DE LECTURE — TRI RAPIDE

Référence (APA 7) : ...
Type de source : article / ouvrage / rapport / ...
Question de recherche : ...
Méthodologie : ...
Résultats clés : ...
Citations utiles (avec pages) : ...
Limites identifiées par l'auteur : ...
Lien avec ma problématique : ...
```

### Conseils d'utilisation

- **Remplir la fiche immédiatement après la lecture**, pas trois jours plus tard quand on a tout oublié
- **Une fiche = une source.** Ne pas mélanger plusieurs articles sur la même fiche
- **Le champ « Lien avec ma problématique »** est le plus important : c'est lui qui permet de savoir pourquoi on cite cette source dans le mémoire
- **Stocker les fiches** dans un dossier dédié (Zotero permet d'attacher des notes à chaque référence)
- **Numéroter les fiches** pour s'y retrouver facilement quand la bibliographie dépasse 30 sources

## Récapitulatif — Les étapes d'une bonne recherche documentaire

```
1. Définir sa problématique et ses mots-clés (FR + EN)
2. Lancer une première recherche large (Google Scholar, Cairn)
3. Identifier 2-3 articles clés récents
4. Appliquer la technique de la boule de neige
5. Évaluer chaque source avec la grille CRAAP
6. Remplir une fiche de lecture pour chaque source retenue
7. Organiser ses sources dans un gestionnaire (Zotero recommandé)
8. Vérifier qu'on a un mix équilibré : sources FR + EN, récentes + fondatrices,
   articles + ouvrages + données
```
